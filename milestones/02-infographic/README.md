# Milestone 2: The Infographic

**Duration:** ~10 minutes  
**Workspace:** D2 — `workspace-naive/` (still no skills)  
**Goal:** Generate a visual data artifact. Evaluate whether "polished" means "trustworthy."

## What Happens

Same naive workspace. Same data. One more prompt: generate a standalone HTML infographic.

The agent will produce a real deliverable — a visual, shareable HTML report with charts and styled layout. The lesson is that professional presentation can make an analysis look more confident than it is.

## Setup

No additional setup from Milestone 1. Same workspace, same data loaded.

## Suggested Prompts

See `prompts.md` for the milestone prompt sequence.

## Opening the Output

1. Ask the agent to save the HTML to a file (or copy it from the chat)
2. Save as `infographic-naive.html` in `workspace-naive/`
3. Open in VS Code integrated browser (right-click > Open with Live Server) or drag to Chrome
4. View it as a stakeholder would — not as a developer

## What to Watch For

- Does the report disclose where any number comes from?
- Are data quality issues mentioned anywhere?
- Does the chart confidently show an average that includes the Suzzallo anomaly without flagging it?
- Is there an assumptions section?
- If you sent this to your dean, would they have reason to trust or question it?

## Transition to Milestone 3

The infographic is genuinely impressive. It's also undefendable. That tension — between surface quality and analytical trustworthiness — is what Milestone 3 resolves.
