# Spotlight Walkthrough

Adds a Shepherd.js guided tour to an HTML data report. The tour walks a reader through the data story in narrative order, with contextual explanations at each stop.

## When to Use

- After generating an HTML data report (use after `html-infographic-builder`)
- User asks to "add a tour," "create a guided walkthrough," or "make the report self-presenting"
- Milestone 4 final infographic — the spotlight tour is part of the deliverable
- Any standalone HTML report that will be viewed without the analyst present to explain it

## Inputs

- An existing HTML data report (generated by `html-infographic-builder` or similar)
- Optional: a narrative arc (what story should the tour tell?)
- Optional: audience type (determines tour step emphasis)

## Process

1. Read the HTML report and identify 5-8 key sections to feature in the tour
2. Determine narrative order: the tour should tell the data story from context → insight → implication
3. Inject Shepherd.js from CDN into the `<head>`:
   ```html
   <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/shepherd.js@13/dist/css/shepherd.css">
   <script src="https://cdn.jsdelivr.net/npm/shepherd.js@13/dist/js/shepherd.min.js"></script>
   ```
4. Add `data-tour-step="[step-name]"` attributes to the target elements in the HTML
5. Generate the Shepherd.js tour initialization script at the bottom of `<body>`
6. Add a "Start Tour" button in the report header

## Tour Step Structure

Each tour step should have:
- `title`: brief label (5 words max)
- `text`: 2-3 sentences explaining what this section shows and why it matters
- `attachTo`: the element with the matching `data-tour-step` attribute
- `buttons`: Back / Next (final step: "Done")

Recommended step sequence for a data report:
1. **Report header** — introduce the scenario and time range
2. **Headline KPIs** — the top-line numbers and their significance
3. **Primary finding chart** — the most important visual
4. **Anomaly or quality note** — surface a flagged issue and how it was handled
5. **Confidence scores panel** — explain what confidence scores mean and how to read them
6. **Audience-specific insight** — the most relevant finding for this reader
7. **Assumptions log** — where to go if you need to validate the analysis
8. **Call to action** — what the reader should do with this information

## Shepherd.js Initialization Template

```javascript
const tour = new Shepherd.Tour({
  useModalOverlay: true,
  defaultStepOptions: {
    classes: 'shadow-md bg-white',
    scrollTo: true,
    cancelIcon: { enabled: true },
    buttons: [
      { text: 'Back', action: tour.back },
      { text: 'Next', action: tour.next }
    ]
  }
});

tour.addStep({
  id: 'step-name',
  title: 'Step Title',
  text: 'Explanation of what this section shows and why it matters.',
  attachTo: { element: '[data-tour-step="step-name"]', on: 'bottom' }
});

// ... additional steps

// Replace last step's buttons to say "Done" instead of "Next"
tour.start();
```

## Output

- Modified version of the input HTML file with Shepherd.js injected
- "Start Tour" button added in the report header
- Tour runs automatically on page load if `?tour=1` query param is present (optional)
- File renamed: `[original-name]-with-tour.html`
